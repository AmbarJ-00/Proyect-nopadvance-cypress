# Proyect-nopadvance-Cypress

## Objetivo
Este proyecto tiene como objetivo hacer un recorrido sobre la aplicacion noPadvance para verificar su funcionamiento, hacer un analisis de su estructura y detectar oportunidades de mejora, asi emmpleando las habilidades y conocimientos sobre la automatizacion de pruebas disponibles en mi persona.

## Integraciones
* Cypress.
* Gherkin.
* GIT.


## Patrones de diseño
- Page object model(POM): es un patrón de diseño que crea un repositorio/carpeta de objetos para los elementos de la interfaz de usuario de una aplicación web.
Idea central: Cada página web de la aplicación se modela como una clase Java (u otro lenguaje).


## Historias de usuario
Las historias de usuario son redactadas en gherkin para que sean accesibles y flexibles para cualquier tipo de lector que quiera entender en base a que tipo de usuario, necesidades y funcionalidades se trabajo en este proyecto de automatizacion.
